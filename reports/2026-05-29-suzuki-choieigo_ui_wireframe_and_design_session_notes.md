# ちょい英語 ミニブック投稿システム — UI 制作セッション記録

> Claude（Anthropic）との対話で進めた、**ワイヤーフレーム整備 → デザイン適用**の2フェーズにわたる UI 制作セッションの統合記録です。チーム共有・レビュー用。

---

## 1. セッションの目的

添付の仕様ワイヤーフレーム PDF（A-P1 〜 A-U2、全20画面）を起点に、コーディング着手可能な状態に近い UI 雛形を整備すること。

進め方は次の2フェーズ。

- **フェーズ1：ワイヤーフレーム整備** — 1画面ずつ作るのではなく、UI 部品とページ類型に分解し、コンポーネントカタログ1枚＋ページ類型4枚に集約
- **フェーズ2：デザイン適用** — 既存の消費者向けアプリ画面（参考PNG）からトンマナを抽出し、フェーズ1で作った4枚のページモックアップに反映

### 2-2. ページ類型の使い分け

20画面はパーツ共通だが、ページ単位の構成は4類型に集約できる。各類型の代表1枚を用意した。

- **詳細／編集型**（`page_mockup.html`）— タブ＋多数のパネル＋下部追従の保存バー。最もパーツ種類が多い
- **一覧型**（`page_list_members.html`）— テーブル主役、フィルタ行＋ページャ
- **フォーム型**（`page_form_course_new.html`）— ラベル＋入力のフィールド行＋ラジオ群＋3ボタン保存バー
- **プロフィール型**（`page_profile.html`）— 表示のみ KV パネル、アクション無し

## 5. フェーズ1 議論ポイント — 「ワイヤーフレームだけでコーディングできるか」

ターン3でのユーザー問いに対する Claude の回答（要約）。チームでの認識合わせに有用なため記載。

### カバーできていること（視覚層）

- 全コンポーネントとその状態（インベントリ）
- 主要4類型のページレイアウト
- デザイントークン体系

### 静的カンプでは原理的に描けないこと（追加で必要）

1. **動的な状態** — 空っぽ（0件）、読み込み中、通信失敗、保存成功、文字あふれ、ロールによる出し分け
2. **インタラクションと画面遷移** — hover/focus/クリック、モーダル開閉、バリデーション発火タイミング、画面間遷移
3. **ビジネスルール** — 外部公開ロックによる実効公開範囲の格下げ、1会員1団体制約、会員削除時の投稿ユニット所有権の自動移譲、メディア供給モード切替時の削除挙動など
4. **デザイントークンの言語化** — 色・余白・タイプスケールを再利用可能な形に整理

### 結論

> 「全コンポーネント（インベントリ済）＋ページ類型を数枚＋挙動とルールのメモ」が揃えば、安心してコーディングに入れる状態になる。今ある5枚で、その3点セットのうち1.5点ぶんが手元にある感覚。

## 6. フェーズ2 議論ポイント — デザイン適用時の判断記録

参考PNGから何を抽出し、どう投稿システムに翻訳したか。レビュー時の参考に。

### 6-1. 配色の決定（参考PNGからの実測値）

サンプリングツール（Pillow）で実画像から拾い、推測値ではなく実測値で確定。

| 用途 | 色 | 由来 |
|------|------|------|
| CTAプライマリ | `#007AFF` | ログイン画面の「ログイン」ボタン |
| ブランド濃紺 | `#0459B7` | ロゴ本体・統計数値・サイドバー強調色 |
| シアン（アクセント） | `#00A9F4` | タブ下線・キャラクター色 |
| お気に入り黄 | `#FFC400` | 一覧画面のハート（参考流用は最小限） |
| 状態色 緑／赤／橙 | 個別調整 | 完了・エラー・警告 |
| 背景 | `#EEF2F7` | カードが浮き立つ淡いブルーグレー |

### 6-2. フォント・アイコン

- 本文：**Noto Sans JP**（IBM Plex Sans JP から変更）。基準サイズ 13px → 14px、行間 1.6 → 1.75 に拡大して可読性優先
- モノスペース：**Roboto Mono** に最小限使用（ID・タイムスタンプのみ）
- アイコン：絵文字・記号（🔍 ⭳ 🎙 ⚠ ⋮⋮ ★ ▾ ‹ › ▲▼）を**全廃**し、**Material Symbols Rounded** に統一。サイドバー項目・バッジ内アイコン・テーブル内ドラッグハンドル・ページャ矢印・フォームのセレクト矢印まで含めて統一適用

### 6-3. 投稿システム向けの判断

参考PNGは消費者向け学習アプリのため、そのまま移植せず以下の判断を入れた。

- **使いやすさ最優先** — 角丸・影・余白は採用したが、アニメーションや派手な装飾は加えていない
- **状態色の明確化** — 必須／任意ラベルは色＋形で区別、削除ゾーンは赤系で危険を明示
- **操作のスキャナビリティ** — サイドバーのアクティブ項目は青のピル背景、タブはシアン下線で現在地が一目で分かる
- **黄ハートは限定使用** — 参考PNGで多用されているお気に入り黄は、管理画面では「TOPICMAKER運営」フラグなど特別な意味を持つ箇所に限定し、一般操作には使わない

### 6-4. 実装の効率化

- フェーズ1で全ページが同一CSS（`:root` 変数共通）だったため、フェーズ2は1枚分のCSSを作って4枚に一括適用できた
- HTML側の構造・日本語ラベル・ドメイン用語は一切変更せず、視覚層のみの差し替えで完結
- 描画検証は Playwright で各ページのフルページスクリーンショットを取得して目視確認（Material Symbols が描画されているか、配色が意図通りか）

## 8. 制作上のメモ

### 共通

- 5ファイルすべてで CSS の `:root` 変数を共通化（フェーズ2適用後の現行値：`--primary` `--brand` `--cyan` `--ink` `--line` `--bg` 等）。色を変える際は変数を差し替えるだけで全画面に反映可能
- ロール出し分けは3パターン実装済み（団体管理者：フルメニュー／会員・団体一般会員：団体管理ブロック非表示）
- 「黒い開発バナー」はユーザー指示により全ページで非表示
- `wireframe_inventory.html` は意図的にモノクロのまま残置（パーツ一覧としての中立性を維持）

### フェーズ2 で変わった点

- フォント：`IBM Plex Sans JP` + `IBM Plex Mono` → `Noto Sans JP` + `Roboto Mono`（CDN参照）
- アイコン：絵文字・記号の仮置き → `Material Symbols Rounded`（CDN参照）
- 角丸／影／配色：モノクロのフラット表現 → 白カード＋淡いブルーグレー背景＋ソフトな影＋ブランド配色
